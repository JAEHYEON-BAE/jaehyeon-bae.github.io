복잡한 작업 흐름을 분석하고, 누구나 직관적으로 사용할 수 있는 도구로 구현하는 개발자 배재현입니다.

단순히 기능을 구현하는 것보다, 실제 사용자가 겪는 비효율과 불편을 발견하고 이를 안정적으로 동작하는 도구로 바꾸는 과정에 관심이 많습니다.

---

### 소프트웨어를 바라보는 관점

대학 첫 프로그래밍 강의에서 들었던 "프로그래밍의 본질은 추상화"라는 말은 이후 개발을 바라보는 중요한 기준이 되었습니다.

제가 생각하는 추상화는 다음에 가깝습니다.

- 사람이 매번 기억해야 하는 맥락을 줄이는 것
- 반복 작업을 프로그램 내부로 옮기는 것
- 복잡한 상태와 예외 상황을 시스템이 책임지도록 만드는 것
- 사용자는 단순한 동작만으로 원하는 결과를 얻도록 만드는 것
- 내부 복잡성은 감추되, 결과의 정확성과 안정성은 유지하는 것

이 관점은 데스크톱 앱, 하드웨어 연동 프로젝트, AI 워크플로우 설계 모두에서 반복적으로 드러났습니다.

---

### 강점

**AI 기반 개발** — Claude Code, Codex 등 AI 코딩 에이전트를 코드 생성 도구가 아닌 개발 워크플로우 협업 도구로 활용합니다. slash commands, custom hooks, 자동 commit 흐름을 구성해 개발 루틴을 더 안정적이고 반복 가능하게 만들어왔습니다.

**실제 동작하는 도구 제작** — 아이디어를 배포 가능한 형태까지 완성하는 데 강점이 있습니다. GitHub Actions 기반 릴리즈 파이프라인, 실제 사용자, 그리고 실사용 환경에서만 드러나는 문제까지 다뤄왔습니다.

**작업 흐름 단순화** — 프로세스가 복잡해지는 지점을 관찰하고 개선합니다. 비개발자가 공연 현장에서 사용하는 DMX 조명 컨트롤러, AI 작업 과정에서 느낀 불편에서 출발한 Markdown 에디터, 단일 에이전트 출력의 약점을 보완하는 멀티 에이전트 리뷰 시스템이 그 예시입니다.

**실사용 환경 중심 신뢰성** — 관찰 가능한 상태, 재현 가능한 오류, 예외 상황에서의 복구 가능성을 중요하게 생각합니다. 개발 환경과 실제 환경의 차이야말로 가장 흥미로운 문제가 숨어 있는 곳입니다.
